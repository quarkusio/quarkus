### Usage

- Add this extension to deploy Quarkus REST endpoints as AWS Lambda behind API Gateway (REST API type).
- Your existing REST endpoints work as-is — no code changes needed.
- API Gateway translates HTTP requests to Lambda events and responses.

### Deployment

- Build with `./mvnw package` — produces a deployment package in `target/function.zip`.
- Deploy using SAM CLI, AWS CDK, or the AWS Console.
- Use `manage.sh` (generated by Quarkus) for quick deployment.

### Testing

- Use `@QuarkusTest` — test REST endpoints normally with REST Assured.
- Lambda event translation is transparent.

### Common Pitfalls

- This is for API Gateway REST API — for HTTP API type, use `quarkus-amazon-lambda-http`.
- Cold start times can be significant — consider native image or SnapStart for better startup.
- Do NOT use WebSockets or streaming responses — Lambda is request/response only.
