
### Usage

Add the extension — Swagger UI is available at `/q/swagger-ui` in dev mode with no configuration needed. It serves the OpenAPI spec generated by `smallrye-openapi` (included transitively).

### Making It Available in Production

By default, Swagger UI is only included in dev/test mode. To include it in production builds:

```properties
quarkus.swagger-ui.always-include=true
```

### Customization

```properties
# Change the UI path (relative to non-application root, default: /q/)
quarkus.swagger-ui.path=swagger-ui

# UI behavior
quarkus.swagger-ui.doc-expansion=list     # none, list, or full
quarkus.swagger-ui.deep-linking=true
quarkus.swagger-ui.display-operation-id=true
quarkus.swagger-ui.display-request-duration=true
quarkus.swagger-ui.try-it-out-enabled=true

# Theme
quarkus.swagger-ui.theme=flattop          # feeling-blue, flattop, material, monokai, muted, newspaper, outline

# Title and footer
quarkus.swagger-ui.title=My API Documentation
quarkus.swagger-ui.footer=Copyright 2024

# Operations sorting
quarkus.swagger-ui.operations-sorter=alpha   # alpha, method
quarkus.swagger-ui.tags-sorter=alpha
```

### Multiple API Specs

Serve multiple OpenAPI specs in the top bar selector:

```properties
quarkus.swagger-ui.urls.petstore=https://petstore.swagger.io/v2/swagger.json
quarkus.swagger-ui.urls.local=/openapi
quarkus.swagger-ui.urls-primary-name=local
```

### OAuth Configuration

Pre-configure OAuth for "Try it out":

```properties
quarkus.swagger-ui.oauth-client-id=my-client
quarkus.swagger-ui.oauth-app-name=My App
quarkus.swagger-ui.oauth-use-pkce-with-authorization-code-grant=true
quarkus.swagger-ui.persist-authorization=true
```

### Common Pitfalls

- **Dev mode only by default**: Swagger UI is NOT included in production builds unless `always-include=true` is set. This is intentional for security.
- **Requires `smallrye-openapi`**: The UI needs an OpenAPI spec to display. If no REST endpoints exist, the UI will be empty. The `smallrye-openapi` extension is included transitively.
- **Path is relative to non-application root**: The default URL is `/q/swagger-ui`, not `/swagger-ui`. The `/q/` prefix is the non-application root path.
- **`quarkus.swagger-ui.enable` is deprecated** — use `quarkus.swagger-ui.enabled` instead (added `d` suffix). The old property will be removed.
